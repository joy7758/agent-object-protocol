# AEP-0010: v1.1 DSSE Optional Profile

Envelope Packaging for Evidence (No Mandatory Verification)

- Status: Accepted
- Authors: joy7758 (maintainers may append)
- Created: 2026-03-04
- Target release: v1.1.0 (MINOR)
- Related: AEP-0009 (Accepted), AEP-0007 (Accepted)
- Scope: Schema + fixtures + CI gates (no runtime, no mandatory verification)

---

## 1. Abstract

This AEP introduces an optional DSSE envelope profile for AOP evidence
in v1.1.0.

It adds:

- a DSSE envelope schema that carries AOP evidence payloads
- positive and negative fixtures with CI gate coverage
- interoperability alignment with in-toto attestations and Sigstore
  envelope usage

Signature verification remains optional in v1.1.0.

---

## 2. Motivation

AOP v1.0.0 stabilizes evidence artifacts and profile validation,
including digest binding via `SHA-256(JCS(json))`.

To align with broader supply-chain ecosystems, evidence packaging should
support DSSE envelopes:

- DSSE signs arbitrary payloads and binds message type + content.
- in-toto attestation v1 defines envelope packaging via DSSE.
- Sigstore bundle formats can carry attestations as DSSE content.

v1.1.0 adds this as an additive optional profile without changing
v1.0 frozen core semantics.

---

## 3. Scope and non-goals

### In scope (v1.1.0)

- New schema:
  - `schemas/profiles/dsse-envelope.profile.schema.json`
- New fixtures:
  - `examples/v1.1/dsse/positive/*`
  - `examples/invalid/v1.1/dsse/*`
- CI gates for:
  - envelope shape validation
  - payload base64 decode
  - JSON parseability for JSON payload types
  - payload validation against `schemas/aop-evidence.schema.json`

### Non-goals

- Mandatory signature verification and key-management policy
- Rekor/transparency-log integration
- Full Sigstore bundle support

---

## 4. Proposed artifacts

### 4.1 DSSE envelope schema profile

Add:

- `schemas/profiles/dsse-envelope.profile.schema.json`

Minimal fields:

- `payloadType` (recognized MIME-like type for AOP evidence payload)
- `payload` (base64-encoded payload bytes)
- `signatures[]` (optional in v1.1, if present each entry requires `sig`)

### 4.2 AOP payloadType for v1.1

v1.1 fixtures MUST support:

- `application/vnd.aop.evidence+json`

Additional payload types MAY be added in later MINOR releases.

---

## 5. CI binding gates (v1.1)

For each positive DSSE fixture:

1. envelope validates against DSSE profile schema
2. payload decodes from base64
3. decoded payload parses as JSON for `application/*+json`
4. decoded payload validates against `schemas/aop-evidence.schema.json`

Negative fixtures MUST be rejected, including:

- invalid base64 payload
- unsupported payloadType
- invalid JSON payload
- payload that is JSON but fails AOP evidence schema

---

## 6. Conformance integration

Introduce Level 8 (optional) in `CONFORMANCE.md` after DSSE gates and
fixtures are complete:

- Level 8 requires Level 7 plus DSSE envelope packaging validation and
  payload binding checks.

---

## 7. Compatibility

- v1.1.0 is a MINOR additive release.
- No frozen v1.0 fields or semantics are removed.
- DSSE support is optional and reflected as an additive conformance
  level.
